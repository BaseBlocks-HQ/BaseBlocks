Pages use one ordered block document. Add columns when content should sit side by side, and add page tabs when readers need to switch between related views.

## Columns

Insert a columns block from the `/` menu, then add and reorder content inside each column. Columns stack vertically on smaller screens so the page remains readable.

## Page tabs

Use the tabs command to wrap the current document in a tabbed page. Each tab contains its own block document and can be renamed, reordered, added, or removed.

## Spacing

Use normal paragraphs and document spacing instead of empty layout rows. This keeps page structure predictable across editing, viewing, and export.
