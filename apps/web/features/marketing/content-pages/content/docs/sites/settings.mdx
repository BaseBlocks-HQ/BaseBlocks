Access site settings from the editor sidebar. Basic site actions are also available from the site card on the team dashboard.

## General

**Name** — The display name for the site. Shown in the dashboard, the editor, and optionally in the site navigation. Can be changed at any time.

**Slug** — The path segment that identifies this site in the URL:

```
https://your-team.baseblocks.dev/[site-slug]
```

<Callout type="info">
  The site slug is chosen when the site is created and cannot currently be changed from Site Settings.
</Callout>

**Favicon** — Upload the small icon used for the published site's browser tab.

## Header and navigation

Choose whether to show the logo and site name in the published sidebar and search in the always-visible header.

## Custom domains

Add a hostname such as `docs.example.com` from the Custom Domain section. BaseBlocks reports the required DNS record and verification status. After DNS is configured and verified, the published site opens at the root of that domain with TLS managed automatically.

## Visibility

Controls who can view the live (published) site. Four options:

| Mode | Who can view |
|---|---|
| Public | Anyone with the URL |
| Private | Signed-in team members only |
| Link-only | Anyone with the exact URL (not discoverable) |
| Password | Anyone with the access code |

See [Site Visibility](/docs/publishing/visibility) for details on each mode.

## Deleting a site

Site deletion is available from the site controls. Deleting a site removes its pages, legacy and new-editor content, documents, assets, domains, access sessions, and search entries permanently.

<Callout type="error">
  Site deletion is permanent and cannot be undone. There is no recovery path after deletion.
</Callout>
