Accédez aux paramètres du site depuis le panneau **Site** du rail de l'éditeur. Les actions de base sont également disponibles depuis la carte du site dans le tableau de bord.

## Général

**Nom** — Le nom d'affichage du site. Affiché dans le tableau de bord, l'éditeur et optionnellement dans la navigation du site. Peut être modifié à tout moment.

**Identifiant** — Le segment de chemin qui identifie ce site dans l'URL :

```
https://votre-equipe.baseblocks.dev/[identifiant-site]
```

<Callout type="info">
  L'identifiant du site est choisi à sa création et ne peut pas être modifié actuellement depuis les paramètres.
</Callout>

**Métadonnées du site** — Configurez le titre du navigateur, la description, les mots-clés, le favicon et l'image Open Graph utilisés par les pages publiées.

## En-tête et navigation

Choisissez d'afficher ou non l'en-tête du site publié, le logo, le nom du site, la recherche dans l'en-tête et le fil d'Ariane. La navigation latérale peut être ouverte ou fermée par défaut. Son style se configure dans le panneau **Navigation** du rail.

## Domaines personnalisés

Ajoutez un nom d'hôte comme `docs.exemple.com` dans la section Domaine personnalisé. BaseBlocks indique l'enregistrement DNS requis et son état de vérification. Une fois le DNS configuré et vérifié, le site publié s'ouvre à la racine du domaine avec TLS géré automatiquement.

## Visibilité

Contrôle qui peut voir le site publié. Quatre options :

| Mode | Qui peut voir |
|---|---|
| Public | Toute personne avec l'URL |
| Privé | Membres de l'équipe connectés uniquement |
| Par lien | Toute personne avec l'URL exacte (non découvrable) |
| Mot de passe | Toute personne avec le code d'accès |

Voir [Visibilité du site](/docs/publishing/visibility) pour les détails sur chaque mode.

## Supprimer un site

La suppression est disponible depuis les contrôles du site. Elle efface définitivement les pages, les contenus de l'ancien et du nouvel éditeur, les documents, les ressources, les domaines, les sessions d'accès et les entrées de recherche.

<Callout type="error">
  La suppression d'un site est permanente et ne peut pas être annulée. Il n'y a pas de chemin de récupération après la suppression.
</Callout>
