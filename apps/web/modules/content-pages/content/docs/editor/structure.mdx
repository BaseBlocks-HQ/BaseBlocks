Page content is a single ordered tree. Most blocks render directly, while structure blocks contain other blocks.

## Structure blocks

| Type | Contains | Description |
|---|---|---|
| Columns | One or more columns | Groups blocks side by side on desktop and stacks them on mobile. |
| Tabs | One or more tab panels | Keeps related content in one place without creating another page. |
| Spacer | Nothing | Adds vertical whitespace between content blocks. |

## Working with structure

### Adding structure

Open **Structure** in the rail and choose columns, tabs, or a spacer. The block is inserted into the current page content.

### Moving structure

Structure blocks move like every other page block. Drag them up or down the canvas to reorder the page.

### Deleting structure

Deleting a columns or tabs block deletes the nested content inside it.

<Callout type="warn">
  Deleting structure is undoable with `Cmd Z` / `Ctrl Z`, but only within the current session. Reloading the page clears undo history.
</Callout>

## Mobile behavior

Columns collapse to a single column on mobile. Tabs and spacers keep their content order, with spacer heights reduced on smaller screens.
